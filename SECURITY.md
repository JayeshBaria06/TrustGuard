# Security Policy

## Supported Versions

Currently, only the latest version of TrustGuard is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

We take the security of TrustGuard seriously. If you believe you have found a security vulnerability, please report it to us as soon as possible.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to security@trustguard.example.com (placeholder).

Please include the following in your report:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full path of source file(s) related to the manifestation of the issue
- The location of the affected code or a description of the step-by-step process to reproduce the issue
- Any evidence of the vulnerability (e.g., screenshots, exploit code, etc.)

## Security Update Policy

Upon receiving a report, we will:

1. Acknowledge the report within 48 hours.
2. Provide an estimated timeline for a fix.
3. Keep you informed of our progress.
4. Notify the community once a fix is released.
